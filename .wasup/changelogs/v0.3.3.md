## [v0.3.3] - 2026-06-30

### Fixed

- **Runtime memory management** — Fixed `m4_free_value` shallow teardown, corrected function signature, and made `div_u`/`mod_u` nil-safe to prevent leaks and crashes (M1)
- **QBE backend state leaks** — Reset optimization state before each compilation in `qbe_wrap` to prevent stale configuration leaking between builds (M2)
- **QBE non-int boxing** — Track `BoxKind` instead of bare `bool` in `ensureBoxed` for correct boxing of non-integer values (M3)
- **QBE build cache** — Added hash-based runtime object cache keyed by source+target combination instead of file existence check, preventing stale compilations (M4)
- **Parser diagnostics** — Propagated structured diagnostics for lex errors (`p001`) with proper `had_error` tracking (M5)
- **CLI lint output** — Improved lint error format for cleaner diagnostics display (M6)
- **Thread Value tags** — Added dedicated `Value` tags for `thread_handle` and `channel` types (M7)
- **Stdlib cleanup** — Resolved `@constCast` warnings, updated `Io` usage, and fixed `stringBuilder` memory leak (M8)
- **Test runner stability** — Stable exit code capture, improved script portability across macOS and Linux (M9)
- **Thread/channel tests** — Enhanced worker functionality to send multiple values, added thread join and rigorous assertions
- **Benchmark script robustness** — Quoted file paths to handle spaces, ensured deterministic binary selection (Bun, python3)

### Changed

- **`m4_free_value` API** — Signature now accepts boxed `M4Value*` pointer instead of raw value, preventing accidental misuse with unboxed integers
- **Div/mod boxing** — Integer division and modulo now return properly boxed results (or `&m_nil` on divide-by-zero)
- **API documentation** — Clarified boxing semantics for `m4_new_bool`, `m4_new`, `nil-as`, `div`, and `mod` in runtime headers
- **Documentation** — Updated 6 markdown files with v0.3.3 version bumps (`README.md`, `README_ZH.md`, `SPEC.md`, `AGENTS.md`, `skills/m4-coder/SKILL.md`, `skills/m4-coder/reference/SPEC.md`)

- **Skill reference** — Replaced deprecated `io.*` API references with `std.*` equivalents in `skills/m4-coder/SKILL.md`; fixed garbled text and corrected `std.range` examples
- **PURPOSE.md** — Updated for v0.3.3 bug-fix release goals and success criteria
