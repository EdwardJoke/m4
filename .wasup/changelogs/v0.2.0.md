## [v0.2.0] - 2026-05-27

### Added
- QBE native compilation backend — emits QBE SSA IR from the AST and produces native binaries
  - QBE C sources integrated into build.zig as a static library
  - `src/qbe.zig` — QBE IR emitter walking AST nodes
  - `src/qbe_build.zig` — native binary build pipeline (compile + link)
  - Full type lowering: int, float, bool, string, struct, vec
  - String constants and stdlib calls in generated QBE IR
- `--native` CLI flag to emit QBE IR instead of running via bytecode VM
- `m4 build <file>` subcommand to compile directly to native binary
- Minimal C runtime (`src/runtime/m4rt.c`, `m4rt.h`) for compiled program entry/linking
- QBE C API wrapper (`src/runtime/qbe_wrap.c`, `qbe_wrap.h`) for build integration
- String operations in the bytecode VM: lexicographic comparison (`<`, `>`, `<=`, `>=`),
  length via `len()` builtin, and character indexing via `s[i]` syntax
- AI context instructions (generated language model guidance)

### Changed
- Documentation synced to match v0.2.0 codebase state:
  - `AGENTS.md` — added QBE pipeline path, new source files, string operations to current state
  - `README.md` — updated CLI flags, project structure with QBE/runtime modules, version numbers
  - `ROADMAP.md` — updated "no separate IR" statement to reflect QBE SSA IR emitter
- Refactored VM comparison logic: `cmpSlow` decomposed into `cmpSlow` → `cmpOrder` → `cmpValues`
- Updated error descriptions (r005, r008, r009) to document string support alongside vectors
