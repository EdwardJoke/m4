## [v0.3.1] - 2026-06-25

### Fixed

- **Integer overflow detection** — Arithmetic operations (add, sub, mul, div, neg) now use Zig's `@*WithOverflow` builtins and return a `r017` runtime error instead of panicking in debug mode or silently wrapping in release mode. Division by `minInt / -1` and negation of `minInt` are also guarded (#4)
- **Global cache LRU fix** — The VM's 4-entry global value cache now properly promotes accessed entries to the front (position 0) on both hits and misses, with position 3 as the eviction candidate. Previously it shifted entries left and inserted at the back, providing negligible performance benefit (#5)
- **Memory management for heap-allocated objects** — `Compiler.deinit()` now frees `FunObj` instances in the chunk constant table; `VM.deinit()` frees `StringBuilderObj` instances from globals and tracks stdlib string allocations for cleanup; stdlib `readln`/`readAll` no longer leak their temporary buffers; QBE runtime gains a `m4_free_value()` function for native binaries (#6)
- **stdlib API compatibility** — Fixed `Allocator.duplicate()` → `dupe()` and added missing allocator parameter to `ArrayList.append()` calls

### Security

- Integer overflow now produces a deterministic runtime error (`r017`) rather than undefined behavior that varies between debug and release build modes
