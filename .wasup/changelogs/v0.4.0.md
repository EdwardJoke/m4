## [v0.4.0] - 2026-07-01

### Added

- **Project scaffolding** — `mein init`/`mein new` command creates a new m4 project directory with `main.m4` entry point and `.wasup/wasup.toml` configuration (M5)
- **Cache management** — `mein clean` command removes `.m4_cache` build artifacts to reclaim disk space (S1)
- **Shell completion** — Bash and Zsh completion scripts for both `m4c` and `mein`; `mein completions bash|zsh [m4c|mein]` command outputs the script (C1)
- **`--version`/`-v` flags** — Added to both `m4c` and `mein` for quick version lookup (C2)
- **CLI polish** — Improved command parsing with unified `-h`/`--help` across all subcommands, alias handling, and friendlier error messages via `printCommandError`

### Changed

- **Codebase restructured** — All compiler `.zig` files moved from `src/` to `src/compiler/`; new `src/toolchain/` directory houses the toolchain binary (M1)
- **Build system** — `build.zig` now produces two binaries: `m4c` (compiler with QBE backend) and `mein` (toolchain manager) (M2)
- **Binary renamed** — `m4` renamed to `m4c` to reflect its compiler-only scope (M3)
- **CLI help updated** — All help text, error messages, and structured help metadata (`HelpInfo`, `VersionInfo`) updated from `m4` to `m4c` (S2)
- **Documentation** — `README.md`, `README_ZH.md`, `SPEC.md`, `AGENTS.md`, and `skills/m4-coder/SKILL.md` updated with renamed binary references (S2)
- **Test scripts** — Binary path updated from `zig-out/bin/m4` to `zig-out/bin/m4c` in `run_tests.sh`, `bench.sh`, `mem_profile.sh`, `m4.sh`, and `hardspeed_concat.sh` (S3)

### Fixed

- **Scanner EOF newline** — Scanner now emits an explicit `NEWLINE` token when `in_line` is true before EOF, making line state consistent
- **VM diagnostic leak** — Fixed diagnostic buffer leak when diag is null by deferring `allocator.free(msg)` only in the non-null path
- **VM struct equality** — Added proper `add_obj` equality and register representation for structs; compares structs in equality table and stores struct objects in a dedicated `.struct_obj` union field instead of reusing `.vec`
- **VM global cache string comparison** — Fixed global cache key comparison to use `std.mem.eql` slices instead of manual pointer/length compare
- **VM string type narrowing** — Tightened string equality to compare only when both operands are string-like (`and` instead of `or`), preventing mixed string/non-string comparisons
- **VM call/register window** — Introduced `arg_count` and `arg_base`, validate register window bounds before advancing frames, move pc update to correct point to prevent overflow
- **VM struct alloc/register** — Adjusted struct allocation and register assignment to use `_obj` and updated `struct_set`/`struct_get_field` casts accordingly

### Removed

- **`m4` binary** — Replaced by two separate binaries: `m4c` (compiler) and `mein` (toolchain)
