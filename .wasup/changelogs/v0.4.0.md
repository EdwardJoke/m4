## [v0.4.0] - 2026-07-01

### Added

- **Project scaffolding** ‚Äî `mein init`/`mein new` command creates a new m4 project directory with `main.m4` entry point and `.wasup/wasup.toml` configuration (M5)
- **Cache management** ‚Äî `mein clean` command removes `.m4_cache` build artifacts to reclaim disk space (S1)
- **Shell completion** ‚Äî Bash and Zsh completion scripts for both `m4c` and `mein`; `mein completions bash|zsh [m4c|mein]` command outputs the script (C1)
- **`--version`/`-v` flags** ‚Äî Added to both `m4c` and `mein` for quick version lookup (C2)
- **CLI polish** ‚Äî Improved command parsing with unified `-h`/`--help` across all subcommands, alias handling, and friendlier error messages via `printCommandError`
- **Dedicated character literal parsing** ‚Äî Added specific prefix function for character literals, extracting the single character from the token lexeme into a `char_lit` AST node

### Changed

- **Codebase restructured** ‚Äî All compiler `.zig` files moved from `src/` to `src/compiler/`; new `src/toolchain/` directory houses the toolchain binary (M1)
- **Build system** ‚Äî `build.zig` now produces two binaries: `m4c` (compiler with QBE backend) and `mein` (toolchain manager) (M2)
- **Binary renamed** ‚Äî `m4` renamed to `m4c` to reflect its compiler-only scope (M3)
- **CLI help updated** ‚Äî All help text, error messages, and structured help metadata (`HelpInfo`, `VersionInfo`) updated from `m4` to `m4c` (S2)
- **Documentation** ‚Äî `README.md`, `README_ZH.md`, `SPEC.md`, `AGENTS.md`, and `skills/m4-coder/SKILL.md` updated with renamed binary references (S2)
- **Test scripts** ‚Äî Binary path updated from `zig-out/bin/m4` to `zig-out/bin/m4c` in `run_tests.sh`, `bench.sh`, `mem_profile.sh`, `m4.sh`, and `hardspeed_concat.sh` (S3)

### Fixed

- **Scanner EOF newline** ‚Äî Scanner now emits an explicit `NEWLINE` token when `in_line` is true before EOF, making line state consistent
- **VM diagnostic leak** ‚Äî Fixed diagnostic buffer leak when diag is null by deferring `allocator.free(msg)` only in the non-null path
- **VM struct equality** ‚Äî Added proper `add_obj` equality and register representation for structs; compares structs in equality table and stores struct objects in a dedicated `.struct_obj` union field instead of reusing `.vec`
- **VM global cache string comparison** ‚Äî Fixed global cache key comparison to use `std.mem.eql` slices instead of manual pointer/length compare
- **VM string type narrowing** ‚Äî Tightened string equality to compare only when both operands are string-like (`and` instead of `or`), preventing mixed string/non-string comparisons
- **VM call/register window** ‚Äî Introduced `arg_count` and `arg_base`, validate register window bounds before advancing frames, move pc update to correct point to prevent overflow
- **VM struct alloc/register** ‚Äî Adjusted struct allocation and register assignment to use `_obj` and updated `struct_set`/`struct_get_field` casts accordingly
- **VM return handling and frame unwinding** ‚Äî Moved decoding of Ax/ret value earlier so registers are read before frame becomes invalid; switched `spawnFn` temporary allocations from VM allocator to stable allocator to prevent use-after-free on error paths
- **Compiler reg_count after expression statements** ‚Äî Recompute highest register slot in use after compiling expression statements to prevent stale `reg_count` from corrupting subsequent compilation

### Removed

- **`m4` binary** ‚Äî Replaced by two separate binaries: `m4c` (compiler) and `mein` (toolchain)
