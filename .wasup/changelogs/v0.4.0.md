## [v0.4.0] - 2026-07-01

### Added

- **Project scaffolding** — `mein init`/`mein new` command creates a new m4 project directory with `main.m4` entry point and `.wasup/wasup.toml` configuration (M5)
- **Cache management** — `mein clean` command removes `.m4_cache` build artifacts to reclaim disk space (S1)
- **Shell completion** — Bash and Zsh completion scripts for both `m4c` and `mein`; `mein completions bash|zsh [m4c|mein]` command outputs the script (C1)
- **`--version`/`-v` flags** — Added to both `m4c` and `mein` for quick version lookup (C2)

### Changed

- **Codebase restructured** — All compiler `.zig` files moved from `src/` to `src/compiler/`; new `src/toolchain/` directory houses the toolchain binary (M1)
- **Build system** — `build.zig` now produces two binaries: `m4c` (compiler with QBE backend) and `mein` (toolchain manager) (M2)
- **Binary renamed** — `m4` renamed to `m4c` to reflect its compiler-only scope (M3)
- **CLI help updated** — All help text, error messages, and structured help metadata (`HelpInfo`, `VersionInfo`) updated from `m4` to `m4c` (S2)
- **Documentation** — `README.md`, `README_ZH.md`, `SPEC.md`, `AGENTS.md`, and `skills/m4-coder/SKILL.md` updated with the renamed binary references (S2)
- **Test scripts** — Binary path updated from `zig-out/bin/m4` to `zig-out/bin/m4c` in `run_tests.sh`, `bench.sh`, `mem_profile.sh`, `m4.sh`, and `hardspeed_concat.sh` (S3)

### Removed

- **`m4` binary** — Replaced by two separate binaries: `m4c` (compiler) and `mein` (toolchain)
