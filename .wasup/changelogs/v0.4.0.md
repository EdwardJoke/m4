## [v0.4.0] - 2026-07-01

### Added

- **Project scaffolding** — `mein init` / `mein new` command creates a new m4 project with `main.m4` and `.wasup/wasup.toml` (M5)
- **Cache management** — `mein clean` command removes build artifacts (`.m4_cache`) (S1)
- **Shell completion scripts** — bash/zsh completions for both `m4c` and `mein`; `mein completions` command generates them (C1)
- **`--version`/`-v` flags** — added to both `m4c` and `mein` binaries (C2)

### Changed

- **Codebase restructured** — All compiler `.zig` files moved from `src/` to `src/compiler/`; new `src/toolchain/` directory for toolchain code (M1)
- **Build system** — `build.zig` now builds both `m4c` (compiler) and `mein` (toolchain) binaries (M2)
- **Binary renamed** — `m4` is now `m4c` (compiler-only binary) (M3)
- **Toolchain separated** — Non-compiler CLI functions extracted into `mein` binary with subcommand dispatch (M4)
- **CLI help updated** — All help text, error messages, and documentation updated to reference `m4c` instead of `m4` (S2, S3)

### Removed

- **`m4` binary** — Replaced by `m4c` (compiler) and `mein` (toolchain)
